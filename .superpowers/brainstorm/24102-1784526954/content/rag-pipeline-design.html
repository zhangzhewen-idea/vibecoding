第四节：RAG 入库、检索与流式回答

参数保持可配置但给出教学默认值；检索过程显式编排，确保可以保存来源引用。

A · 异步文档入库
1. 安全上传≤ 20 MB
扩展名 + MIME 校验
UUID 文件名并落盘
→
2. 解析.md：MarkdownDocumentReader
其余：Apache Tika
提取标题与来源元数据
→
3. 切片TokenTextSplitter
目标 800 tokens
约 100 tokens 重叠
→
4. 向量化qwen3.7-text-embedding
固定 1024 维
批次最多 20 条
→
5. 入 RedisHNSW / COSINE
写入 KB、文档、页码、标题等元数据
状态改为 READY
失败策略：任一步失败均将文档标记为 FAILED，保存可读错误原因；已写入的本次切片按 documentId 回滚，原文件保留，管理员可点击重试。
B · 单知识库 RAG 问答
1. 权限与会话验证用户、会话归属
确认知识库已启用
加载最近 10 轮消息
→
2. 语义检索knowledgeBaseId 精确过滤
Top-K = 6
相似度阈值 = 0.60
→
3. 证据提示词上下文只包含命中切片
指示模型不得把文档内容当指令
无证据时固定拒答
→
4. SSE 流式输出qwen3.7-plus
delta → references → done
完成后保存回答、耗时与引用
为什么显式检索而不直接隐藏在 Advisor 中？
应用层能拿到命中的 Document 列表，从而生成引用卡片、保存相似度、排查检索质量；Spring AI 仍负责模型、文档、切片和向量存储抽象。
引用定位规则
始终包含知识库、文件名和摘要；Markdown 优先显示标题；PDF 只有在 Tika 提供页信息时显示页码，否则不伪造页码。
